lista no ordenadas

se usan parra marcar articulos cuyo orden no importan, solo el que se representen, el ejemplo clasico es una lista de compras, cada lisa desordenada se empieza con un (ul) que es el acronimo de ingles para (unordered list), el conteindo den la lista se compone de (li) (list iterm)

  • leche
  • huevos
  • chorizo
  • harina
  • mezcal

lista ordenadas

las listas ordenadas son lo contrario a las desordenadas, el orden de la lista si importa, esta se relaizan con la etiqueta (ol), e igual sigune unsado (li)

  1. JS
  2. python
  3. rust
  4. java

listas anidadas

la anidacion de listas es casi lo mimso que anidar ciclos o condiciones, lo que hace es que dentro de una listra agregamos otra lista, esto es valido, si es requerido, en seguida tenemos un ejemplo de como seria esto.

  1. Pela el ajo y picarlo en trozos gruesos.
  2. Retira las semillas y el tallo del pimiento, y cortarlo en trozos gruesos.
  3. Mete todos los alimentos en un procesador de alimentos.
  4. Procesa todos los ingredientes hasta conseguir una pasta.
    • Si deseas un hummus "grueso", procésalo corto tiempo.
    • Pica durante más tiempo si se desea obtener un hummus "suave".

cursiva, negrita, subrayado

los elementos con las etiqeutas (b) que hacen el texto en engrita, (i) cursiva y (u) que subraya el texto, no tienen un peso en la semantica, en la estrucutra de la pagina, estos sirgieron en un tiempo en donde CSS no esta tan estandarizado para darle estilo a las pagina, estos elementos se llamas elementos de presetacion, a diferentia con (string) y (em) los cuales si tiene un peso semantico


en HTMl5 se les dio un significado semantico, los cuales son

  • (i): se usa para transmitir el significado que tradicionalemtne se trasmite la italica, extrajerismos, clasificaciones taxonocimcas, conceptos tecnicos..
  • (b): se usa para transmitir el significado el cual puede ser: palabras clave, nombres de productos, frases principales
  • (u): se usa para nombres propios, errores ortograficso...
